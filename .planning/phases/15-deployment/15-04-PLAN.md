**DEPLOY-05: Security Headers**
Run from terminal:
```bash
curl -I https://YOUR-VERCEL-URL/
```

Confirm these headers are present in the response:
- `x-frame-options: DENY` (or SAMEORIGIN)
- `x-content-type-options: nosniff`
- `content-security-policy: ...` (should contain default-src, script-src, etc.)
- `strict-transport-security: max-age=...`
- `referrer-policy: ...`
- `permissions-policy: ...`

**DEPLOY-04: Rate Limiting**
Run from terminal (replace URL with yours):
```bash
for i in $(seq 1 11); do
  echo -n "Request $i: "
  curl -s -o /dev/null -w "%{http_code}" -X POST https://YOUR-VERCEL-URL/login
  echo ""
done
```

Requests 1-10 should return 200 (or 302/405). Request 11 should return **429** (Too Many Requests).

NOTE: If you hit rate limit during testing, wait 60 seconds before retrying or your subsequent tests will also be rate-limited.

**DEPLOY-03: Sentry Error Capture**
1. Open your production URL in a browser
2. Open the browser developer console (F12 -> Console tab)

```javascript
fetch('/api/debug/sentry-example-api').then(r => r.text()).then(console.log)
```
(If this endpoint does not exist, navigate to any page and run `throw new Error("Sentry production test")` in the console.)

4. Go to https://sentry.io -> your project -> Issues tab
5. Confirm a new error event appears (may take 30-60 seconds)
6. Click the error -- confirm the stack trace shows readable source code (not minified)
