<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create Neon database and push schema</name>
  <action>
**Step 1: Create Neon project**
1. Go to https://console.neon.tech
2. Sign up or log in
3. Click "New Project"
4. Name: `baseaim-production` (or similar)
5. Select region closest to your users (e.g., US East for US-based clients)
6. Click "Create Project"

**Step 2: Copy connection strings**
From the Neon dashboard Connection Details panel, copy TWO connection strings:
- **Pooled** (hostname contains `-pooler`): This goes in Vercel as DATABASE_URL
- **Direct** (hostname without `-pooler`): This is for running prisma db push from your local machine

**Step 3: Push schema to production database**
Run this from the project root directory (replace the URL with your DIRECT connection string):

```bash
DATABASE_URL="postgresql://user:pass@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require" npx prisma db push
```

You should see output confirming all models were synced.

**Step 4: Seed admin user**
Run this to create the admin account in production:

```bash
DATABASE_URL="postgresql://user:pass@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require" npx prisma db seed
```

Confirm the seed completed without errors.
  </action>
  <how-to-verify>
1. Neon dashboard shows a project with tables matching prisma/schema.prisma
2. `prisma db push` output shows "Your database is now in sync with your Prisma schema"
3. `prisma db seed` completes without errors
  </how-to-verify>
  <resume-signal>Paste your POOLED DATABASE_URL (the one with -pooler in hostname) and confirm seed succeeded, then type "database ready"</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Create Vercel project and configure all environment variables</name>
  <action>
**Step 1: Generate AUTH_SECRET**
Run locally:
```bash
openssl rand -base64 32
```
Copy the output. This is your AUTH_SECRET.

**Step 2: Create Vercel project**
1. Go to https://vercel.com -> "Add New..." -> "Project"
2. Import your GitHub repository (baseaimclientdashboard)
3. Framework Preset: Next.js (should auto-detect)
4. **IMPORTANT: Do NOT click Deploy yet** -- add environment variables first

**Step 3: Add environment variables**
In the Vercel project settings (Settings -> Environment Variables), add ALL of the following for the **Production** environment scope:

| Variable | Value | Notes |
|----------|-------|-------|
| `DATABASE_URL` | Your Neon POOLED connection string (with `-pooler`) | From Task 1 |
| `AUTH_SECRET` | Output of `openssl rand -base64 32` | Generated in Step 1 |
| `NEXT_PUBLIC_APP_URL` | `https://your-project.vercel.app` | Use your Vercel project URL for now; update later if adding custom domain |
| `STRIPE_SECRET_KEY` | Your live Stripe key (`sk_live_...`) | Stripe Dashboard -> Developers -> API keys -> Live mode |
| `STRIPE_WEBHOOK_SECRET` | `whsec_placeholder` | Temporary -- will be updated in Plan 03 after registering webhook |
| `RESEND_API_KEY` | Your Resend API key | Resend dashboard -> API Keys |
| `RESEND_FROM_EMAIL` | Your verified sender email | Optional -- only if you have a verified domain in Resend |
| `GOOGLE_OAUTH_CLIENT_ID` | Your Google OAuth client ID | Google Cloud Console -> APIs & Services -> Credentials |
| `GOOGLE_OAUTH_CLIENT_SECRET` | Your Google OAuth client secret | Same location as above |
| `GOOGLE_OAUTH_REFRESH_TOKEN` | Your Google OAuth refresh token | See Plan 03 Task 2 if you need to regenerate for production |
| `GOOGLE_DRIVE_ROOT_FOLDER_ID` | The Google Drive folder ID for client documents | From the Drive folder URL |
| `UPSTASH_REDIS_REST_URL` | Placeholder: leave empty for now | Will be configured in Plan 03 |
| `UPSTASH_REDIS_REST_TOKEN` | Placeholder: leave empty for now | Will be configured in Plan 03 |
| `SENTRY_DSN` | Placeholder: leave empty for now | Will be configured in Plan 03 |
| `NEXT_PUBLIC_SENTRY_DSN` | Placeholder: leave empty for now | Will be configured in Plan 03 |
| `SENTRY_ORG` | Placeholder: leave empty for now | Will be configured in Plan 03 |
| `SENTRY_PROJECT` | Placeholder: leave empty for now | Will be configured in Plan 03 |
| `SENTRY_AUTH_TOKEN` | Placeholder: leave empty for now | Will be configured in Plan 03 |
| `SKIP_ENV_VALIDATION` | `true` | **Temporary** -- allows first deploy without all services configured. Remove after Plan 03 completes. |

**IMPORTANT:** Set `SKIP_ENV_VALIDATION=true` for the first deploy so the build passes without Sentry/Upstash/Stripe webhook configured yet. We will remove this after Plan 03 configures all remaining services.

**Step 4: Deploy**
Click "Deploy" in the Vercel dashboard (or trigger a redeploy if it already deployed).

**Step 5: Verify deployment**
1. Wait for build to complete (check Vercel build logs for errors)
2. Visit your Vercel URL (e.g., `https://your-project.vercel.app`)
3. You should see the login page
4. Try logging in with your admin credentials (from seed)
