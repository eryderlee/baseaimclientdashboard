<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Set up Upstash Redis and Sentry</name>
  <action>
**Upstash Redis setup:**
1. Go to https://console.upstash.com
2. Sign up or log in
3. Click "Create Database"
4. Name: `baseaim-ratelimit` (or similar)
5. Select region closest to your Vercel deployment region
6. Type: Regional (not Global -- lower latency, sufficient for rate limiting)
7. After creation, go to the "REST API" tab
8. Copy `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN`

**Sentry setup:**
1. Go to https://sentry.io
2. Sign up or log in
3. Create a new project: Platform = "Next.js"
4. Note your **org slug** (from URL: `sentry.io/organizations/YOUR-ORG/`)
5. Note your **project slug** (the project name you created)
6. Go to Project Settings -> Client Keys (DSN) -> copy the DSN URL
7. Go to Settings -> Developer Settings -> Auth Tokens -> Create New Token
   - Scopes needed: `project:releases`, `org:read`, `project:read`, `project:write`
   - Copy the auth token

**Update Vercel env vars:**
Go to your Vercel project -> Settings -> Environment Variables and update/add:

| Variable | Value |
|----------|-------|
| `UPSTASH_REDIS_REST_URL` | From Upstash REST API tab |
| `UPSTASH_REDIS_REST_TOKEN` | From Upstash REST API tab |
| `SENTRY_DSN` | From Sentry Client Keys |
| `NEXT_PUBLIC_SENTRY_DSN` | Same value as SENTRY_DSN |
| `SENTRY_ORG` | Your Sentry org slug |
| `SENTRY_PROJECT` | Your Sentry project slug |
| `SENTRY_AUTH_TOKEN` | From Sentry Auth Tokens |
  </action>
  <how-to-verify>
1. Upstash dashboard shows your database with 0 keys
2. Sentry dashboard shows your project created
3. All 7 env vars are updated in Vercel dashboard
  </how-to-verify>
  <resume-signal>Type "upstash and sentry configured" to continue</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Register Stripe webhook and publish Google OAuth consent screen</name>
  <action>
**Stripe production webhook:**
1. Go to https://dashboard.stripe.com
2. **IMPORTANT: Switch to Live mode** (toggle in top-left, not Test mode)
3. Go to Developers -> Webhooks -> "Add endpoint"
4. Endpoint URL: `https://YOUR-VERCEL-URL/api/webhooks/stripe` (use the URL from Plan 02)
5. Select events to listen to:
   - `invoice.paid`
   - `invoice.payment_failed`
   - `checkout.session.completed`
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
6. Click "Add endpoint"
7. On the endpoint detail page, click "Reveal" under Signing secret
8. Copy the signing secret (starts with `whsec_`)

**Update STRIPE_WEBHOOK_SECRET in Vercel:**
Go to Vercel -> Settings -> Environment Variables -> Edit `STRIPE_WEBHOOK_SECRET` -> paste the live webhook signing secret.

**Google OAuth consent screen:**
1. Go to https://console.cloud.google.com
2. Navigate to APIs & Services -> OAuth consent screen
3. Check the "Publishing status"
4. If it says "Testing", click "Publish App" -> confirm
5. This ensures your `GOOGLE_OAUTH_REFRESH_TOKEN` does not expire after 7 days

**If you need a NEW refresh token for production:**
1. Go to https://developers.google.com/oauthplayground/
2. Click gear icon -> check "Use your own OAuth credentials"
3. Enter your Client ID and Client Secret
4. Step 1: Select scope `https://www.googleapis.com/auth/drive`
5. Click "Authorize APIs" -> complete the consent flow
6. Step 2: Click "Exchange authorization code for tokens"
7. Copy the `refresh_token` value
8. Update `GOOGLE_OAUTH_REFRESH_TOKEN` in Vercel

**Remove SKIP_ENV_VALIDATION:**
In Vercel -> Settings -> Environment Variables -> **Delete** the `SKIP_ENV_VALIDATION` variable entirely.

**Redeploy:**
Go to Vercel -> Deployments -> click the three dots on the latest deployment -> "Redeploy" (or push a commit to trigger a new build).
  </action>
  <how-to-verify>
1. Stripe webhook endpoint appears in live mode dashboard with status "Enabled"
2. Google OAuth consent screen shows "In production" status
3. SKIP_ENV_VALIDATION no longer exists in Vercel env vars
4. Vercel redeploy succeeds (build passes env validation without SKIP_ENV_VALIDATION)
5. App still loads at public URL after redeploy
  </how-to-verify>
  <resume-signal>Confirm all items above are done and redeploy succeeded, then type "services configured"</resume-signal>
