  1. **Install jspdf-autotable:**

  2. **Add imports to lib/dal.ts** — extend the existing facebook-ads import line:
     ```typescript

     ```

  3. **Add getClientFbCampaigns DAL function** after the existing `getClientFbDailyInsights`:
     - Follow the EXACT same pattern as `getClientFbInsights`: `verifySession()` outside cache, check CLIENT role, lookup adAccountId, lookup facebookAccessToken, wrap `fetchFacebookCampaignInsights` in `unstable_cache`
     - Cache key: `fb-campaigns-${client.id}-${datePreset}`
     - Tags: `[fb-insights-${client.id}]`
     - Revalidate: 21600 (6 hours)
     - Returns `FbCampaignInsight[]` (empty array when not configured)
     - Accepts `datePreset: DatePreset = 'last_30d'` parameter

  4. **Add getClientFbPlatformBreakdown DAL function:**
     - Same pattern as above
     - Calls `fetchFacebookPlatformBreakdown`
     - Cache key: `fb-platform-${client.id}-${datePreset}`
     - Tags: `[fb-insights-${client.id}]`
     - Returns `FbPlatformRow[]`
     - Accepts `datePreset: DatePreset = 'last_30d'` parameter

  5. **Add getClientFbDailyTrend DAL function:**
     - Same pattern but calls existing `fetchFacebookDailyInsights` (which now returns extended fields with reach + outbound_clicks)
     - Cache key: `fb-daily-trend-${client.id}`
     - Tags: `[fb-insights-${client.id}]`
     - Returns `FbDailyInsight[] | null` (null when not configured)
     - No datePreset param — daily trend is always last 30 days (hardcoded in fetchFacebookDailyInsights)

  CRITICAL: verifySession() MUST be called BEFORE the unstable_cache boundary in all three functions. This is a Next.js constraint — cookies/headers are not available inside unstable_cache callbacks.

  Each function wraps with React `cache()` at the outer level, matching existing DAL pattern.
