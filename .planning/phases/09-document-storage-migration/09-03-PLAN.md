  <name>Task 1: Create download proxy route</name>
  <files>app/api/documents/download/[fileId]/route.ts</files>
  <action>
    Create `app/api/documents/download/[fileId]/route.ts` with a GET handler:

    1. Authenticate the request using `auth()` from `@/lib/auth`. Return 401 if no session.

    2. Extract `fileId` from params (async params pattern: `const { fileId } = await params`).

    3. Authorization check: Query `prisma.document.findFirst({ where: { fileUrl: fileId } })` to find the document by its Drive file ID (stored in fileUrl column). Include the client relation with user. Verify that either:
       - The session user owns this document (document.client.userId === session.user.id), OR
       - The session user has ADMIN role
       Return 403 if neither condition is met. Return 404 if document not found.

    4. Call `streamFileFromDrive(fileId)` from `@/lib/google-drive` to get the stream, mimeType, and fileName.

    5. Convert the Node.js ReadableStream to a Web ReadableStream for the Response. Use:
       ```typescript
       const { Readable } = require('stream')
       const webStream = Readable.toWeb(stream)
       ```
       Or use the pattern:
       ```typescript
       return new Response(stream as unknown as ReadableStream, {
         headers: {
           'Content-Type': mimeType,
           'Content-Disposition': `attachment; filename="${encodeURIComponent(fileName)}"`,
           'Cache-Control': 'private, no-cache',
         },
       })
       ```

    6. Wrap in try/catch. On Drive API errors, return 500 with error message. Log the error.

    IMPORTANT: Use `responseType: 'stream'` in the Drive API call (handled by streamFileFromDrive). This streams the file instead of buffering it, which avoids the Vercel 4.5 MB response body limit for buffered responses.
  </action>

    - Route imports streamFileFromDrive from @/lib/google-drive
    - Route returns streaming Response with Content-Disposition header
    - `npm run build` compiles without errors
  </verify>
