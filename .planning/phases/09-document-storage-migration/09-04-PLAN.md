  <name>Task 1: Create migration script and run audit</name>
  <files>scripts/migrate-blob-to-drive.ts</files>
  <action>
    Create `scripts/migrate-blob-to-drive.ts` - a one-time migration script run via `npx tsx scripts/migrate-blob-to-drive.ts`. Follow the same pattern as `prisma/seed-milestones.ts`.

    The script should have three phases:

    **Phase A: Audit**
    - Query all documents from DB
    - Categorize: how many have Vercel Blob URLs (contain 'vercel-storage.com' or 'blob.vercel'), how many have local dev paths ('/uploads/'), how many already have Drive file IDs
    - Log: total documents, by category, total file size
    - NOTE: Research suggests BLOB_READ_WRITE_TOKEN is empty in .env, meaning there may be ZERO real Blob files. The audit confirms this.

    **Phase B: Create Drive Folders for Existing Clients**
    - Query all clients where `driveFolderId IS NULL`
    - For each: call `createClientDriveFolder(client.companyName, client.id)` and update the client record with the returned folder ID
    - Log success/failure per client
    - Continue on individual failures (don't abort the whole script)

    **Phase C: Migrate Blob Files (if any exist)**
    - For each document with a Vercel Blob URL:
      1. Fetch the file from the Blob URL using `fetch()`
      2. Get the client's `driveFolderId`
      3. Upload to Drive using `uploadFileToDrive()`
      4. Update the document's `fileUrl` to the Drive file ID
      5. Log success
    - For documents with local dev paths ('/uploads/'): log them as "local dev files - no migration needed" and optionally clear them or leave as-is
    - Do NOT delete from Blob yet - log a summary at the end

    **Summary output:**
    - Clients: X total, Y folders created, Z failed
    - Documents: X total, Y migrated, Z already on Drive, W local dev (skipped)
    - "Run with --delete-blob flag to remove Vercel Blob files after verifying migration"

    Import prisma from `@/lib/prisma` - use relative path since this is a script. Actually, since tsx can use path aliases with tsconfig, import normally. If path alias doesn't work in scripts, use relative: `import { prisma } from '../lib/prisma'` and `import { createClientDriveFolder, uploadFileToDrive } from '../lib/google-drive'`.

    NOTE: The `import 'server-only'` in google-drive.ts will error when run from a script. Handle this by either:
    - Creating a separate `lib/google-drive-core.ts` without the import (avoid this - too much refactoring)
    - OR: Set the `SKIP_SERVER_ONLY` env var and add a conditional guard in google-drive.ts (avoid this - hacky)
    - OR: Simply remove the `import 'server-only'` temporarily, run the script, then add it back (pragmatic for a one-time script)
    - BEST OPTION: The script can directly instantiate its own Drive client using the same pattern, bypassing the server-only module entirely. Copy the auth setup inline in the script. This is a one-time script - DRY doesn't matter.
  </action>
