<task type="auto">
  <name>Task 2: Create Drive client singleton and helper functions + update Prisma schema</name>
  <files>lib/google-drive.ts, prisma/schema.prisma</files>
  <action>
    1. Create `lib/google-drive.ts` with `import 'server-only'` at the top (same pattern as lib/email.ts). Include:

       - `getDriveClient()`: Returns a singleton `drive_v3.Drive` instance. Uses `GoogleAuth` with credentials from `GOOGLE_SERVICE_ACCOUNT_EMAIL` and `GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY` env vars (NOT full JSON). Apply `privateKey.replace(/\\n/g, '\n')` for newline handling. Scope: `https://www.googleapis.com/auth/drive`. Cache the client instance in module scope (like lib/prisma.ts pattern).

       - `createClientDriveFolder(companyName: string, clientId: string): Promise<string>`: Creates a folder named `{companyName} - {clientId}` in the root folder (from `GOOGLE_DRIVE_ROOT_FOLDER_ID` env var). Returns the folder ID. Throws if no ID returned.

       - `uploadFileToDrive(file: File, folderId: string): Promise<{id: string, name: string, size: number}>`: Converts File to Buffer, creates Readable stream, uploads via `drive.files.create` with `media` parameter. The googleapis library auto-selects resumable upload for large files. Returns file metadata.

       - `streamFileFromDrive(driveFileId: string): Promise<{stream: NodeJS.ReadableStream, mimeType: string, fileName: string}>`: Gets metadata first (name, mimeType), then gets content with `{alt: 'media'}` and `{responseType: 'stream'}`. Cast `response.data` as `unknown as NodeJS.ReadableStream` (known TypeScript inference issue with googleapis).

       - `deleteFileFromDrive(driveFileId: string): Promise<void>`: Calls `drive.files.delete({fileId})`.

       - `listFilesInFolder(folderId: string): Promise<drive_v3.Schema$File[]>`: Lists non-trashed files in folder with fields `id,name,mimeType,size,createdTime,modifiedTime`, ordered by `createdTime desc`, pageSize 100.

       Import types: `import { drive_v3 } from '@googleapis/drive'` and `import { google as googleDrive } from '@googleapis/drive'` and `import { GoogleAuth } from 'google-auth-library'` and `import { Readable } from 'stream'`.

    2. Update `prisma/schema.prisma`: Add `driveFolderId String?` field to the `Client` model, with a comment `// Google Drive folder ID for this client's documents`. Place it after the `address` field.

    3. Run `npx prisma db push` to apply the schema change.

    4. Run `npx prisma generate` to regenerate the Prisma client.

    Note: Keep `fileUrl` column name in Document model unchanged (Decision 4 from research: Option A for simplicity). After migration it will hold Drive file IDs, but the column name stays generic.
  </action>
