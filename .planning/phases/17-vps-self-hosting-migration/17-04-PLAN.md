<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Update external service URLs</name>
  <action>
    Update all external services that reference the old Netlify URL to use https://dashboard.baseaim.co.
  </action>
  <instructions>
    **1. Stripe Webhook URL:**
    - Go to Stripe Dashboard -> Developers -> Webhooks
    - Find the existing webhook endpoint (pointing to old Netlify URL)
    - Either update the URL to `https://dashboard.baseaim.co/api/webhooks/stripe` OR create a new endpoint with the same events (invoice.paid, invoice.payment_failed, customer.subscription.updated, customer.subscription.deleted)
    - If you create a new endpoint: copy the new webhook signing secret and update `STRIPE_WEBHOOK_SECRET` in `/var/www/dashboard/.env.production` on the VPS, then run `pm2 reload ecosystem.config.js --env production`

    **2. Google OAuth Redirect URIs:**
    - Go to Google Cloud Console -> APIs and Services -> Credentials
    - Find the OAuth 2.0 Client ID used by the app
    - Add `https://dashboard.baseaim.co` to Authorized redirect URIs (keep old URL too during transition)
    - Add `https://dashboard.baseaim.co/api/auth/callback/google` if Google OAuth login is used

    **3. Resend (Email):**
    - Verify the sending domain in Resend Dashboard still works (should be fine -- email sending doesn't depend on app URL)

    **4. (Optional) Decommission Netlify:**
    - Once VPS is verified working, you can disable or delete the Netlify site
    - Remove Netlify-specific env vars if any
  </instructions>
  <resume-signal>Type "done" when Stripe webhook URL is updated, Google OAuth redirect URIs include dashboard.baseaim.co</resume-signal>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Full integration verification</name>
  <what-built>Complete VPS migration with all external services configured</what-built>
  <how-to-verify>
    Run through this verification checklist:

    **1. Basic Load + Performance:**
    - Visit https://dashboard.baseaim.co -- page should load in <500ms (check browser DevTools Network tab, look at DOMContentLoaded time)
    - Verify HTTPS (padlock icon in browser)
    - Check security headers: `curl -I https://dashboard.baseaim.co` -- should show Content-Security-Policy, X-Frame-Options, Strict-Transport-Security

    **2. Authentication:**
    - Log in with a test client account -- should work without redirect issues
    - Log in with admin account -- should redirect to /admin
    - Log out and log back in -- session should persist correctly

    **3. Suspense Streaming:**
    - Visit the analytics page -- loading skeletons should appear briefly before data loads (NOT a blank page that suddenly fills in)
    - This confirms `proxy_buffering off` is working in Nginx

    **4. Rate Limiting:**
    - Open a new incognito window
    - Try logging in with wrong credentials rapidly 11+ times within 60 seconds
    - After 10 attempts, should get "Too many requests" (429 response)

    **5. Sentry:**
    - Check Sentry Dashboard for any errors from the VPS deployment
    - If no errors yet, trigger a test by visiting a non-existent page or checking pm2 logs for any captured exceptions

    **6. Stripe Webhooks (if testable):**
    - In Stripe Dashboard -> Webhooks, check the new endpoint shows "Delivered" status (not failed)
    - Or send a test webhook event from Stripe Dashboard

    **7. Google Drive (if testable):**
    - Try uploading or viewing a document in the dashboard to verify Google Drive integration

    **8. Zero-Downtime Deploy Test:**
    - Make a trivial code change (e.g., a comment) and push to main
    - Watch GitHub Actions -- build should succeed
    - While deploy is in progress, refresh the dashboard -- should remain accessible (zero downtime)
  </how-to-verify>
  <resume-signal>Type "approved" if all checks pass. If any issues, describe what failed.</resume-signal>
</task>
