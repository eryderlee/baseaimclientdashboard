<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Provision VPS software and app directory</name>
  <action>
    SSH into the VPS and run these commands. The user should execute them in order.
  </action>
  <instructions>
    SSH into your VPS and run the following commands:

    **1. Audit current state:**
    ```bash
    cat /etc/os-release
    free -h
    nproc
    node --version 2>/dev/null || echo "No Node.js"
    nginx -v 2>/dev/null || echo "No Nginx"
    ```
    Note the output -- needed to confirm Node 20+ and available RAM.

    **2. Install NVM and Node.js 20:**
    ```bash
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
    source ~/.bashrc
    nvm install 20
    nvm alias default 20
    node --version  # Must be >= 20.9
    ```

    **3. Install PM2 globally:**
    ```bash
    npm install -g pm2
    pm2 install pm2-logrotate
    ```

    **4. Install Nginx and Certbot:**
    ```bash
    sudo apt update
    sudo apt install -y nginx certbot python3-certbot-nginx
    ```

    **5. Create app directory and PM2 log directory:**
    ```bash
    sudo mkdir -p /var/www/dashboard
    sudo chown $USER:$USER /var/www/dashboard
    sudo mkdir -p /var/log/pm2
    sudo chown $USER:$USER /var/log/pm2
    ```

    **6. Create .env.production on VPS:**
    Copy all env vars from your Netlify dashboard. Create the file:
    ```bash
    nano /var/www/dashboard/.env.production
    ```

    Required variables (copy values from Netlify env vars):
    ```
    NODE_ENV=production
    DATABASE_URL=postgresql://...
    AUTH_SECRET=...
    AUTH_TRUST_HOST=true
    NEXT_PUBLIC_APP_URL=https://dashboard.baseaim.co
    STRIPE_SECRET_KEY=sk_live_...
    STRIPE_WEBHOOK_SECRET=whsec_...
    RESEND_API_KEY=re_...
    RESEND_FROM_EMAIL=...
    GOOGLE_OAUTH_CLIENT_ID=...
    GOOGLE_OAUTH_CLIENT_SECRET=...
    GOOGLE_OAUTH_REFRESH_TOKEN=...
    GOOGLE_DRIVE_ROOT_FOLDER_ID=...
    UPSTASH_REDIS_REST_URL=https://...
    UPSTASH_REDIS_REST_TOKEN=...
    SENTRY_DSN=https://...
    NEXT_PUBLIC_SENTRY_DSN=https://...
    ```

    CRITICAL: `AUTH_TRUST_HOST=true` is NEW -- not in Netlify. Required for Auth.js v5 behind Nginx.

    ```bash
    chmod 600 /var/www/dashboard/.env.production
    ```

    **7. Configure Nginx:**
    ```bash
    sudo nano /etc/nginx/sites-available/dashboard.baseaim.co
    ```

    Paste this config:
    ```nginx
    upstream nextjs_app {
        server 127.0.0.1:3000;
        keepalive 64;
    }

    server {
        listen 80;
        server_name dashboard.baseaim.co;
        return 301 https://$host$request_uri;
    }

    server {
        listen 443 ssl http2;
        server_name dashboard.baseaim.co;

        # SSL certs will be added by Certbot
        # ssl_certificate     /etc/letsencrypt/live/dashboard.baseaim.co/fullchain.pem;
        # ssl_certificate_key /etc/letsencrypt/live/dashboard.baseaim.co/privkey.pem;

        # Disable proxy buffering -- required for Next.js Suspense streaming
        proxy_buffering off;

        # Static assets with immutable caching
        location /_next/static/ {
            proxy_pass http://nextjs_app;
            proxy_cache_valid 200 1y;
            add_header Cache-Control "public, max-age=31536000, immutable";
        }

        # Main app proxy
        location / {
            proxy_pass         http://nextjs_app;
            proxy_http_version 1.1;
            proxy_set_header   Upgrade         $http_upgrade;
            proxy_set_header   Connection      "upgrade";
            proxy_set_header   Host            $host;
            proxy_set_header   X-Real-IP       $remote_addr;
            proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header   X-Forwarded-Proto $scheme;
            proxy_set_header   X-Forwarded-Host  $host;
            client_max_body_size 10m;
        }
    }
    ```

    Enable the site:
    ```bash
    sudo ln -s /etc/nginx/sites-available/dashboard.baseaim.co /etc/nginx/sites-enabled/
    sudo nginx -t
    sudo systemctl reload nginx
    ```

    **8. Point DNS to VPS (temporary for SSL cert):**
    Before Certbot can issue a cert, `dashboard.baseaim.co` must resolve to the VPS IP.

    In your DNS provider:
    - Lower TTL to 60 seconds (if possible)
    - Change the A record for `dashboard.baseaim.co` to point to VPS IP

    Wait for DNS propagation (check with `dig dashboard.baseaim.co`).

    **9. Issue SSL certificate:**
    ```bash
    sudo certbot --nginx -d dashboard.baseaim.co
    sudo certbot renew --dry-run  # Verify auto-renewal works
    ```

    **10. Configure GitHub Actions secrets:**
    Go to your GitHub repo -> Settings -> Secrets and variables -> Actions.
    Add these repository secrets:
    - `VPS_HOST` -- your VPS IP address
    - `VPS_USER` -- your SSH username on the VPS
    - `VPS_SSH_KEY` -- full private SSH key content (including `-----BEGIN...` headers)
    - `SENTRY_AUTH_TOKEN` -- from Sentry Dashboard
    - `SENTRY_ORG` -- your Sentry org slug
    - `SENTRY_PROJECT` -- your Sentry project slug
    - `NEXT_PUBLIC_SENTRY_DSN` -- your Sentry DSN

    **11. Trigger first deploy:**
    Push the code changes from Plans 17-01 and 17-02 to main. GitHub Actions will build and deploy automatically.

    After deploy completes, verify PM2 is running:
    ```bash
    pm2 list                    # Should show baseaim-dashboard instances
    pm2 logs baseaim-dashboard  # Check for startup errors
    ```

    Set up PM2 startup persistence:
    ```bash
    pm2 save
    pm2 startup systemd  # Run the command it outputs with sudo
    ```
  </instructions>
