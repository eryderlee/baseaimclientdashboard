<task type="auto">
  <name>Task 1: Create GitHub Actions deploy workflow</name>
  <files>.github/workflows/deploy.yml</files>
  <action>
    Create `.github/workflows/deploy.yml` (create the `.github/workflows/` directory first).

    Use a SINGLE job (not separate build/deploy jobs) to avoid artifact transfer between jobs. The SCP and SSH steps run in the same job after the build step.

    The workflow structure:

    1. **Trigger:** On push to `main` branch only.

    2. **Build steps:**
       - `runs-on: ubuntu-latest`
       - Checkout code with `actions/checkout@v4`
       - Setup Node.js 20 with `actions/setup-node@v4` and `cache: 'npm'`
       - `npm ci`
       - `npm run build` with these env vars:
         - `SKIP_ENV_VALIDATION: '1'` (build doesn't have runtime secrets)
         - `SENTRY_AUTH_TOKEN: ${{ secrets.SENTRY_AUTH_TOKEN }}`
         - `SENTRY_ORG: ${{ secrets.SENTRY_ORG }}`
         - `SENTRY_PROJECT: ${{ secrets.SENTRY_PROJECT }}`
         - `NEXT_PUBLIC_APP_URL: 'https://dashboard.baseaim.co'`
         - `NEXT_PUBLIC_SENTRY_DSN: ${{ secrets.NEXT_PUBLIC_SENTRY_DSN }}`

    3. **Prepare standalone artifact** (run steps in the same job, after build):
       ```bash
       cp -r public .next/standalone/public
       cp -r .next/static .next/standalone/.next/static
       ```
       These copies are required because Next.js standalone output does NOT include `public/` or `.next/static/` by default. After these copies, the `.next/standalone/` directory is fully self-contained.

    4. **Verify artifact integrity** (run step, after prepare):
       ```bash
       test -f .next/standalone/server.js && echo "server.js OK"
       test -d .next/standalone/public && echo "public/ OK"
       test -d .next/standalone/.next/static && echo ".next/static/ OK"
       ```
       This step must fail the workflow if any of the three checks fail.

    5. **Transfer artifact to VPS** using `appleboy/scp-action@v0.1.7`:
       - `host: ${{ secrets.VPS_HOST }}`
       - `username: ${{ secrets.VPS_USER }}`
       - `key: ${{ secrets.VPS_SSH_KEY }}`
       - `source: '.next/standalone/'`
       - `target: '/var/www/dashboard/_deploy'`
       - `overwrite: true`

    6. **SSH to VPS to swap artifact and reload** using `appleboy/ssh-action@v1.0.3`:
       ```bash
       export NVM_DIR="$HOME/.nvm"
       [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"

       cd /var/www/dashboard

       # Swap deployed artifact into place
       rm -rf .next/standalone
       mkdir -p .next
       mv _deploy/.next/standalone .next/standalone

       # Clean up temp deploy dir
       rm -rf _deploy

       # Zero-downtime reload (cluster mode rolling restart)
       # PM2 uses ecosystem.config.js already present at /var/www/dashboard/
       # (placed there during initial VPS provisioning in Plan 17-03)
       pm2 reload ecosystem.config.js --env production
       ```

       IMPORTANT: Do NOT copy ecosystem.config.js from the artifact. PM2 uses the committed copy at `/var/www/dashboard/ecosystem.config.js` that is placed once during VPS provisioning (Plan 17-03). This prevents split source-of-truth between git and deployed artifact.

    7. **Required secrets** (document in YAML comments at top of file):
       - `VPS_HOST` -- VPS IP address
       - `VPS_USER` -- SSH username (e.g., deploy or root)
       - `VPS_SSH_KEY` -- Full private SSH key content
       - `SENTRY_AUTH_TOKEN` -- For source map upload
       - `SENTRY_ORG` -- Sentry organization slug
       - `SENTRY_PROJECT` -- Sentry project slug
       - `NEXT_PUBLIC_SENTRY_DSN` -- Sentry DSN for client-side
  </action>
