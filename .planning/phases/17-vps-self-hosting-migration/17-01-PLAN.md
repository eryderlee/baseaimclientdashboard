
    Wait -- that would create a circular import. Instead, restructure the export pattern. Assign the auth wrapper to a const, then export it both as default and as the named `proxy` export:

    ```typescript

    export default proxyHandler
    export { proxyHandler as proxy }
    ```

    This satisfies Next.js 16's expectation of a named `proxy` export while keeping the `auth()` wrapper and all routing logic identical.

    Summary of changes from original middleware.ts:
    - `export default auth(async (req) => { ... })` becomes `const proxyHandler = auth(async (req) => { ... })`

    - Add `export { proxyHandler as proxy }`
    - The `config` export stays exactly the same
    - All imports stay exactly the same
    - All routing/rate-limiting logic stays exactly the same
