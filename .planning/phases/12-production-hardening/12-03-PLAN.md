  <files>middleware.ts</files>
  <action>
    1. Install packages: `npm install @upstash/ratelimit @upstash/redis`

    2. Modify `middleware.ts` to add rate limiting BEFORE existing auth routing logic:
       - Import `Ratelimit` from `@upstash/ratelimit` and `Redis` from `@upstash/redis`
       - Create Redis client: `const redis = new Redis({ url: process.env.UPSTASH_REDIS_REST_URL!, token: process.env.UPSTASH_REDIS_REST_TOKEN! })`
       - Create rate limiter: `const ratelimit = new Ratelimit({ redis, limiter: Ratelimit.slidingWindow(10, "60 s") })`
       - Define auth routes to rate limit: `["/login", "/reset-password", "/api/auth"]`

    3. Inside the existing `auth((req) => { ... })` handler, BEFORE the existing routing logic:
       - Check if pathname starts with any auth route
       - If auth route: extract IP from `req.headers.get("x-forwarded-for")` (split on comma, take first) or fallback to "127.0.0.1" (Note: `req.ip` may not exist in all runtimes; use x-forwarded-for header which Vercel always sets)
       - WRAP rate limit check in try/catch:
         ```
         try {
           const { success } = await ratelimit.limit(`auth:${ip}`);
           if (!success) {
             return new NextResponse("Too many requests. Please try again later.", {
               status: 429,
               headers: { "Retry-After": "60" },
             });
           }
         } catch (e) {
           console.error("Rate limit check failed:", e);
           // Fall through on rate limiter failure -- don't block on infrastructure issues
         }
         ```

    4. Keep ALL existing auth routing logic unchanged after the rate limit check.

    5. Update the matcher config to also match API routes needed for rate limiting. Current matcher excludes api routes. Change to:
       - Keep existing matcher pattern but ensure `/api/auth` routes are NOT excluded. The current regex `(?!api|_next/static|...)` excludes all /api/ routes from middleware.
       - Add a SECOND matcher entry: `"/api/auth/:path*"` to include auth API routes.
       - Final matcher: `['/((?!_next/static|_next/image|favicon\\.ico|.*\\.png$|.*\\.ico$).*)', '/api/auth/:path*']`
       - WAIT: Actually the current matcher `(?!api|...)` excludes ALL api routes. But the auth routing logic only cares about /dashboard, /admin, /login. The rate limiting on /api/auth won't fire because the matcher excludes /api. We need to update the matcher to include /api/auth paths.
       - Simplest fix: Change the negative lookahead from `(?!api|` to `(?!api/(?!auth)|` -- this excludes /api/* EXCEPT /api/auth/*. Or better: just use a more explicit matcher array.

    IMPORTANT: Wrap rate limit in try/catch. If Upstash is down, requests must still work.
    IMPORTANT: Do NOT change the existing auth routing logic (lines checking isLoggedIn, userRole, redirects). Only ADD rate limiting before it.
    IMPORTANT: The `auth()` wrapper from next-auth must remain as the default export.
