Add isDemo flag and demoStableId to Client model, update all admin DAL queries with conditional isDemo filtering, create static demo FB data constants, and add demo short-circuit logic to client-facing FB functions.

Purpose: This is the data isolation and schema foundation for the entire demo environment. Without isDemo filtering, demo clients would leak into the real admin's view. Without static FB data, demo clients' analytics pages would show "not configured" empty states — a visible indicator that the data is fake.

```typescript
import { DEMO_ADMIN_EMAIL, DEMO_FB_INSIGHTS, DEMO_FB_CAMPAIGNS, DEMO_FB_DAILY_TREND } from '@/lib/demo-data'

// Internal helper: returns true if the calling admin is the demo admin account
async function resolveClientIsDemoFilter(): Promise<boolean> {
  const { userId } = await verifySession()
  const user = await prisma.user.findUnique({
    where: { id: userId },
    select: { email: true },
  })
  return user?.email === DEMO_ADMIN_EMAIL
}
```
