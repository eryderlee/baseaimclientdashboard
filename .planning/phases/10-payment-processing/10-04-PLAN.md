**`startSubscription(formData: FormData)`** — Admin-only action:

Zod schema:
```typescript
const StartSubscriptionSchema = z.object({
  clientId: z.string().min(1),
  monthlyAmount: z.number().positive('Amount must be positive'),
  description: z.string().default('Monthly Retainer'),
  currency: z.string().default('aud'),
})
```

Steps:
1. Verify admin session via verifySession()
2. Parse formData: clientId, monthlyAmount (use parseFloat), description, currency
3. Validate with StartSubscriptionSchema
4. Look up client with user and subscriptions (same pattern as createInvoice)
5. If no client, return { success: false, error: 'Client not found' }
6. Check if client already has an active subscription:
   ```typescript
   const existing = client.subscriptions.find(s => s.stripeSubscriptionId && s.status === 'active')
   if (existing) return { success: false, error: 'Client already has an active subscription' }
   ```
7. Get or create Stripe customer — same lazy-creation pattern as createInvoice (look up stripeCustomerId from subscriptions, create if missing, upsert Subscription record)
8. Create Stripe Price on-the-fly (so admin can set any amount without pre-configuring in Stripe):
   ```typescript
   const price = await stripe.prices.create({
     unit_amount: Math.round(monthlyAmount * 100), // convert to cents
     currency,
     recurring: { interval: 'month' },
     product_data: { name: description },
   })
   ```
9. Create Stripe Subscription:
   ```typescript
   const stripeSubscription = await stripe.subscriptions.create({
     customer: stripeCustomerId,
     items: [{ price: price.id }],
     // Stripe will immediately invoice and charge for the first month
   })
   ```
10. Upsert local Subscription record with all Stripe IDs:
    ```typescript
    await prisma.subscription.upsert({
      where: { clientId },
      update: {
        stripeSubscriptionId: stripeSubscription.id,
        stripePriceId: price.id,
        status: stripeSubscription.status, // 'active' or 'trialing'
        currentPeriodEnd: new Date((stripeSubscription as any).current_period_end * 1000),
      },
      create: {
        clientId,
        stripeCustomerId,
        stripeSubscriptionId: stripeSubscription.id,
        stripePriceId: price.id,
        status: stripeSubscription.status,
        currentPeriodEnd: new Date((stripeSubscription as any).current_period_end * 1000),
      },
    })
    ```
11. revalidatePath for `/admin/clients/${clientId}/invoices` and `/dashboard/billing`
12. Return { success: true }

**`cancelSubscription(formData: FormData)`** — Admin-only action:

Steps:
1. Verify admin session
2. Get clientId from formData
3. Look up Subscription where clientId matches and stripeSubscriptionId is not null
4. If not found, return { success: false, error: 'No active subscription found' }
5. Cancel in Stripe — cancel at period end (client keeps access for remainder of paid period):
   ```typescript
   await stripe.subscriptions.update(subscription.stripeSubscriptionId!, {
     cancel_at_period_end: true,
   })
   ```
6. Update local Subscription: `status: 'cancelling'` (still active until period end, but marked for cancellation)
7. revalidatePath for invoices and billing pages
8. Return { success: true }

Note: When the period actually ends, the `customer.subscription.deleted` webhook (added in Task 2) will set status to 'inactive'.

**In `lib/dal.ts`**, add a new cached DAL function:

```typescript
export const getAdminClientSubscription = cache(async (clientId: string) => {
  const { userRole } = await verifySession()
  if (userRole !== 'ADMIN') throw new Error('Unauthorized')

  return prisma.subscription.findFirst({
    where: { clientId },
    orderBy: { createdAt: 'desc' },
  })
})
```
