**In app/api/webhooks/stripe/route.ts**, create the webhook handler:

```typescript
import { NextRequest, NextResponse } from 'next/server'
import { stripe } from '@/lib/stripe'
import { prisma } from '@/lib/prisma'
import { sendPaymentConfirmationEmail } from '@/lib/email'
import Stripe from 'stripe'
```

Export async function POST(req: NextRequest):
1. **Read raw body:** `const body = await req.text()` — CRITICAL: must be text(), NOT json()
2. **Get signature:** `const signature = req.headers.get('stripe-signature')` — return 400 if missing
3. **Verify event:** Wrap in try/catch: `const event = stripe.webhooks.constructEvent(body, signature, process.env.STRIPE_WEBHOOK_SECRET!)` — return 400 on StripeSignatureVerificationError
4. **Handle event types:**

   For `invoice.paid`:
   - Extract `stripeInvoice = event.data.object as Stripe.Invoice`
   - Find local invoice: `prisma.invoice.findFirst({ where: { stripeInvoiceId: stripeInvoice.id } })`
   - If not found, log warning and return 200 (idempotency — might be external invoice)
   - If already PAID, return 200 (idempotency — duplicate event)
   - Update: `prisma.invoice.update({ where: { id: localInvoice.id }, data: { status: 'PAID', paidAt: new Date() } })`
   - Look up client with user for email: `prisma.client.findUnique({ where: { id: localInvoice.clientId }, include: { user: { select: { email: true, name: true } } } })`
   - Send email (fire-and-forget): `sendPaymentConfirmationEmail({ clientName: client.user.name || client.companyName, email: client.user.email, invoiceNumber: localInvoice.invoiceNumber, amount: localInvoice.amount, currency: localInvoice.currency, paidDate: new Date().toLocaleDateString() })`

   For `invoice.payment_failed`:
   - Extract stripeInvoice from event.data.object
   - Find local invoice by stripeInvoiceId
   - If not found or already OVERDUE, return 200 (idempotency)
   - Update: `prisma.invoice.update({ where: { id: localInvoice.id }, data: { status: 'OVERDUE' } })`

5. **Always return 200:** `return NextResponse.json({ received: true })` — even for unhandled event types. Wrap the handler logic in try/catch; on error, log it but still return 200 to prevent Stripe retry storms.

**In app/api/invoices/[invoiceId]/urls/route.ts**, create GET handler:

Export async function GET(req: NextRequest, { params }: { params: Promise<{ invoiceId: string }> }):
1. Verify session (import auth from '@/lib/auth')
2. Await params, get invoiceId
3. Look up invoice in DB: `prisma.invoice.findUnique({ where: { id: invoiceId } })`
4. If not found, return 404
5. Verify authorization: session user must be admin OR the client who owns the invoice (check via client.userId === session.user.id)
6. If no stripeInvoiceId on invoice, return `{ hosted_invoice_url: null, invoice_pdf: null }`
7. Retrieve from Stripe: `const stripeInvoice = await stripe.invoices.retrieve(invoice.stripeInvoiceId)`
8. Return `{ hosted_invoice_url: stripeInvoice.hosted_invoice_url, invoice_pdf: stripeInvoice.invoice_pdf }`

Do NOT store these URLs in DB — they expire. Always retrieve on-demand.
