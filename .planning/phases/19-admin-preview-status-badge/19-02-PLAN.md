1. Update the `ClientData` interface in `components/admin/client-analytics-table.tsx`:

2. Add a "Setup" column to the table in `ClientAnalyticsTable`:
   - Add `<TableHead>Setup</TableHead>` after the existing "Status" column header.
   - In each row, add a cell that renders a badge:
     ```
     <TableCell>
       <Badge
         variant={client.setupComplete ? 'default' : 'outline'}
         className={client.setupComplete
           ? 'bg-emerald-100 text-emerald-700 hover:bg-emerald-100'
           : 'border-amber-300 text-amber-600'
         }
       >
         {client.setupComplete ? 'Setup Complete' : 'Setting Up'}
       </Badge>
     </TableCell>
     ```
   - Use emerald for complete (positive/green), amber outline for in-progress (matches the preview banner color family).

3. Update `app/admin/page.tsx` to compute `setupComplete` for each client:
   - The page already calls `getAllClientsWithMilestones()` which returns clients with their milestones array (each milestone has `status` and `order` fields).
   - When mapping clients to the `ClientData` shape passed to `ClientAnalyticsTable`, add:
     ```
     setupComplete: client.milestones
       .filter(m => m.order <= 6)
       .every(m => m.status === 'COMPLETED')
     ```
   - This checks if all milestones with order 1-6 are COMPLETED. If a client has fewer than 6 milestones (edge case), `.every()` on an empty array returns true — add a guard: `client.milestones.filter(m => m.order <= 6).length >= 6 && client.milestones.filter(m => m.order <= 6).every(m => m.status === 'COMPLETED')`.
   - No new DB queries needed — `getAllClientsWithMilestones()` already selects `status` and `order` on milestones.
